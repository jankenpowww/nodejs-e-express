#03 - Na prática

Rodando um arquivo JavaScript

Anteriormente vimos como o terminal do NodeJS, que interpreta JavaScript fora do navegador, pasmem, roda mesmo JavaScript. À partir daqui, podemos deduzir que o Node consegue rodar arquivos JavaScript, e ele realmente consegue.

Para isso, precisaremos definir a pasta dentro do terminal onde os arquivos .js estarão pelo prompt de comando, a primeira vista, e abrir ele no terminal. Em seguida, abrir a pasta como projeto em algum editor de código. Concebemos então um arquivo chamado 'ola-mundo.js', que rodará nosso pequeno script dentro do Node.


console.log("Olá, mundo!")

Perceba que o NodeJS não é nada mais nada menos do que foi dito inúmeras vezes até aqui: uma ferramenta que permite rodar JavaScript fora da web, para qual foi originalmente projetado. E com o JavaScript rodando fora do navegador, podemos utilizar a linguagem para construir outros tipos de aplicações, muito além das interações do front-end de um site, da mesma forma com que fazemos com outras linguagens. Então, fora outras particularidades de uso que são diferentes quando utilizamos JavaScript em navegadores, o uso do JavaScript no Node é a mesma coisa.

Com o arquivo criado e o programinha clássico que vai escrever "Olá, mundo!" pronto, podemos abrir o terminal e executar o arquivo escrevendo 'node' + o nome inteiro do arquivo .js! O resultado é algo como...


C:\Users\eu\Desktop\ola-mundo> node ola-mundo.js
Olá, mundo!

O NodeJS, que é um interpretador de JavaScript, realmente interpretou um arquivo JavaScript. Ser redundante as vezes pode ser necessário para entender as coisas.
