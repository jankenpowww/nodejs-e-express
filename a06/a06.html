#06 - Introdução ao Express

Express!

Agora que conhecemos de forma sucinta o intuito do uso do Node para o JavaScript no back-end e como funciona o módulo http que lida com as requisições e respostas entre o cliente e o servidor, vamos conhecer um framework que melhora e adiciona mais funcionalidades para o back-end dos nossos projetos futuros: o Express.

Express ou ainda Express.js, é um framework de desenvolvimento back-end baseado em JavaScript que, como já se espera, é utilizado para lidar com a parte back-end de aplicações, utilizando JavaScript. Se tratando de um framework, ele traz muitas melhorias e funcionalidades interessantes da qual o modulo 'http', visto anteriormente, não aborda de forma tão simples ou funcional, o que permite a construção de aplicações, sobretudo aplicações web, de forma muito prática.



Node Package Manager

Diferentemente do módulo 'http', que é um recurso nativo do Node, o Express precisa ser instalado como uma dependência antes de ser utilizado. Foi dito anteriormente à respeito do npm, ou Node Package Manager, que é o gerenciador de pacotes oficial do NodeJS responsável por gerir bibliotecas e módulos externos para projetos, e que vai ser utilizado para instalar o Express na nossa máquina.

Com o terminal aberto na pasta do projeto, damos um npm init. Ele pedirá, via terminal, uma série de informações a respeito do próprio projeto, que serão consolidadas dentro de um arquivo .json referente às informações do projeto em si, e também de todos os pacotes e módulos que estamos utilizando e suas informações, que estarão presentes dentro deste arquivo também (você pode simplesmente digitar npm init -y para só criar o .json, sem muita burocracia também xD). Para este passo, é necessário que tenha um arquivo chamado 'package.json' nele. Este arquivo .json é muito importante para as instruções futuras, mas por enquanto, é bom ter isso em mente.

Com o arquivo package.json na pasta do projeto, vamos instalar o Express finalmente entrando com o seguinte comando abaixo:


npm install express --save

Note que a flag --save é opcional e o motivo é bem simples: com '--save', você está informando ao npm que deve instalar a dependência, nesse caso o Express, de forma com que todos os arquivos dos módulos do framework sejam adicionados em uma pasta no diretório do nosso projeto. Esses arquivos, ou melhor dizendo, módulos, serão adicionados em uma pasta chamada 'node_modules', e nele, todos os arquivos/módulos que compõe o framework. Note que, com os exemplos anteriores onde modularizamos nossas funções e chamamos em outro arquivo, acontece o mesmo aqui se você verificar cada arquivo-fonte dos módulos do Express. Lembrando que, para ter o diretório com os arquivos de uma dependência que baixamos via npm no nosso projeto, precisamos ter o arquivo package.json no diretório, uma vez que ele reflete automaticamente todas essas adições e informações do projeto durante o desenvolvimento. É possível utilizar dependências sem salvar os arquivos-fonte, propriamente dizendo (?).
